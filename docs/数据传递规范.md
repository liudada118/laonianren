# 肌少症/老年人评估系统 — 数据传递规范

## 一、整体架构

```
┌────────────────────────┐
│ 前端 React (Vite)       │
│ pages/assessment/*.jsx  │
└──────────┬─────────────┘
           │ WebSocket(19999) + HTTP(19245)
           ▼
┌────────────────────────┐
│ BackendBridge.js        │
│ (单例，全局共享)         │
└──────────┬─────────────┘
           │
           ▼
┌────────────────────────┐
│ Node.js 后端             │
│ serialServer.js          │
│ - 串口通信 + 数据采集     │
│ - SQLite 存储             │
│ - 算法调度                │
└──────────┬─────────────┘
           │ subprocess / bridge.py
           ▼
┌────────────────────────┐
│ Python 算法层             │
│ app/algorithms/*.py      │
│ - 数据分析                │
│ - 报告数据生成            │
└────────────────────────┘
```

## 二、统一数据传递流程（六步法）

所有评估模块（握力、起坐、静态足垫、步态）**必须遵循**以下六步流程：

### 第1步：setActiveMode — 设置设备模式

```javascript
await backendBridge.setActiveMode(mode);
// POST /setActiveMode  Body: { mode }
```

| 模式 | 值 | 激活传感器 | 对应评估 |
|------|---:|-----------|---------|
| 左手握力 | 11 | HL | 握力（左手） |
| 右手握力 | 12 | HR | 握力（右手） |
| 双手握力 | 1 | HL + HR | 握力（双手） |
| 起坐评估 | 3 | sit + foot1 | 坐垫+脚垫 |
| 静态站立 | 4 | foot1 | 静态足垫 |
| 步态评估 | 5 | foot1~foot4 | 步道 |

> **重要**：mode 决定了后端 `activeSampleType`，直接影响数据库存储的 `sample_type` 字段，以及后续报告查询的匹配条件。

### 第2步：startCol — 开始数据采集

```javascript
await backendBridge.startCol({
  name: patientInfo?.name || '未知',       // 患者姓名
  assessmentId: assessmentIdRef.current,   // 唯一ID，如 "standing_1709987654321"
  date: new Date().toISOString().split('T')[0],  // 日期
  colName: 'xxx_assessment',              // 采集名称
});
// POST /startCol  Body: { name, assessmentId, date, colName }
```

**assessmentId 命名规范**：`{类型}_{时间戳}`
- 握力：`grip_${Date.now()}`
- 起坐：`sitstand_${Date.now()}`
- 静态站立：`standing_${Date.now()}`
- 步态：`gait_${Date.now()}`

### 第3步：实时数据接收（WebSocket）

后端通过 WebSocket(19999) 推送实时传感器数据：

```javascript
// 手套数据
backendBridge.on('leftHandData', (arr256) => { ... });
backendBridge.on('rightHandData', (arr256) => { ... });

// 脚垫数据
backendBridge.on('foot1Data', (arr4096) => { ... });
backendBridge.on('foot2Data', (arr4096) => { ... });
backendBridge.on('foot3Data', (arr4096) => { ... });
backendBridge.on('foot4Data', (arr4096) => { ... });

// 坐垫数据
backendBridge.on('sitData', (arr1024) => { ... });
```

| 传感器 | 数据长度 | 矩阵尺寸 | 波特率 |
|--------|---------|----------|--------|
| 手套 HL/HR | 256 | 16×16 | 921600 |
| 坐垫 sit | 1024 | 32×32 | 1000000 |
| 脚垫 foot1~4 | 4096 | 64×64 | 3000000 |

### 第4步：endCol — 结束数据采集

```javascript
await backendBridge.endCol();
// GET /endCol
```

结束后，后端将缓冲区数据 flush 到 SQLite 数据库。

### 第5步：getReport — 请求生成报告

```javascript
// 等待数据完全写入
await new Promise(r => setTimeout(r, 500));

// 调用对应的报告生成接口
const resp = await backendBridge.getXxxReport({
  timestamp: new Date().toISOString(),
  assessmentId: assessmentIdRef.current,
  // ...其他参数
});
```

| 评估类型 | 前端方法 | 后端接口 | 额外参数 |
|---------|---------|---------|---------|
| 握力 | `getGripReport()` | `POST /getHandPdf` | leftAssessmentId, rightAssessmentId, collectName |
| 起坐 | `getSitStandReport()` | `POST /getSitAndFootPdf` | collectName |
| 静态站立 | `getStandingReport()` | `POST /getDbHeatmap` | fps, threshold_ratio |
| 步态 | `getGaitReport()` | `POST /getFootPdf` | collectName, body_weight_kg |

### 第6步：渲染报告 + 持久化

```javascript
// 统一判断逻辑
if (resp?.data?.render_data) {
  setPythonResult(resp.data.render_data);

  // 保存到 AssessmentContext（统一格式）
  completeAssessment('类型名',
    { completed: true, reportData: resp.data.render_data },  // report
    { pythonResult: resp.data.render_data }                   // data
  );

  setPhase('report');
} else {
  throw new Error('后端未返回报告数据');
}
```

## 三、后端响应统一格式

所有报告接口返回相同的外层结构：

```javascript
{
  code: 0,             // 0=成功, 1=失败
  data: {
    date: "2024-01-15",
    timestamp: 1705315845123,
    render_data: {     // ← 核心报告数据，结构因评估类型不同
      // ...
    }
  },
  msg: "success"
}
```

## 六、错误处理规范

```javascript
try {
  await backendBridge.endCol();
  await new Promise(r => setTimeout(r, 500));
  const resp = await backendBridge.getXxxReport({ ... });

  if (resp?.data?.render_data) {
    // 成功路径
    setPythonResult(resp.data.render_data);
    completeAssessment(type, { completed: true, reportData: resp.data.render_data }, { pythonResult: resp.data.render_data });
  } else {
    throw new Error('后端未返回报告数据');
  }
} catch (e) {
  console.error('报告生成失败:', e);
  setAnalysisError(e.message || '报告生成失败');
} finally {
  setAnalyzing(false);
  setShowCompleteDialog(true);
}
```

## 七、数据库存储格式

**表名**：`matrix`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 自增主键 |
| data | TEXT | JSON 字符串，包含传感器数据 |
| timestamp | INTEGER | Unix 时间戳 |
| date | TEXT | 日期/采集名称 |
| select | TEXT | 选中的传感器数组 |
| name | TEXT | 患者名称 |
| assessment_id | TEXT | 评估唯一ID |
| sample_type | TEXT | 类型码：1=握力, 3=起坐, 4=静态站立, 5=步态 |

**data 字段 JSON 格式示例**：
```json
{
  "foot1": [4096个数值],
  "foot2": [4096个数值],
  "sit": [1024个数值],
  "HL": [256个数值]
}
```
