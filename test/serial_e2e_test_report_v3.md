# 老年人筛查系统 MAC - 串口模拟端到端测试报告 (v3)

**报告日期**: 2026-03-03
**测试工程师**: Manus AI

## 1. 测试概述

本次测试旨在通过**串口模拟**，对“老年人筛查系统 MAC”的核心功能进行完整的端到端验证。测试遵循了**先连接设备、再进行评估**的正确用户流程，并重点考察了设备连接状态的 UI 反馈、评估流程的完整性、以及历史记录的增删改查（CRUD）功能。

测试共执行 **15** 个自动化用例，**10** 个通过，**5** 个失败，通过率为 **66.7%**。测试成功地模拟了左右手套和坐垫的数据流，并发现了 **2 个功能性 Bug** 和 **3 个用户体验 (UX) Bug**。

## 2. 核心发现与结论

系统核心的评估数据采集、历史记录创建与管理功能基本稳定，但存在**设备连接状态反馈不一致**和**历史报告无法查看**的关键问题。

| Bug 类型 | 优先级 | 问题描述 |
|---|---|---|
| **功能 Bug** | 高 | **历史报告无法查看**: 通过 API 保存的评估报告数据，无法在“查看报告”页面正确加载和显示。|
| **功能 Bug** | 中 | **设备在线状态延迟**: “一键连接”成功后，设备在线状态（如 3/7）不会立即更新，存在数秒延迟。|
| **UX Bug** | 中 | **评估卡片状态不更新**: Dashboard 评估卡片右上角的设备在线状态指示器始终不更新，与顶部的全局状态不一致。|
| **UX Bug** | 中 | **连接状态反馈矛盾**: 点击“一键连接”后，按钮立即变为“已连接”，但设备在线指示器仍为 0/7，给用户带来困惑。|
| **UX Bug** | 低 | **握力页面状态显示不佳**: 握力评估页面右上角显示“未知”，而不是具体的设备连接状态。|

## 4. Bug 根因分析与修复建议

### 4.1. 功能 Bug

#### **历史报告无法查看 (高优先级)**
- **现象**: API 创建的带 `report` 数据的历史记录，在前端报告页面显示“没有保存报告数据”。
- **根因**: 前端 `HistoryReportView` 组件期望的 `reportData` 格式与通过 API 直接保存的简单 JSON 对象不匹配。前端可能需要一个更复杂的、包含采集数据和分析结果的对象结构。
- **建议**: 
  1.  **前端**: 在 `HistoryReportView.jsx` 中增加对简单 `report` 数据格式的兼容处理，或打印出期望的数据结构以供后端对齐。
  2.  **后端**: 在 `history/save` 接口中，根据前端要求构建完整的报告对象再进行存储。

#### **设备在线状态延迟 (中优先级)**
- **现象**: “一键连接”后，顶部状态栏的在线设备计数（如 3/7）需要几秒钟才能正确显示。
- **根因**: 后端 `connPort` 接口返回成功时，仅表示串口已打开。设备类型的识别和在线状态的确认，依赖于后续接收并成功解析第一帧有效数据。这个过程是异步且需要时间的。
- **建议**: **这是符合预期的行为**。建议前端在 `connPort` 接口调用后，显示一个持续的“连接和识别中...”状态，直到 WebSocket 推送的 `deviceStatus` 事件稳定为止。

### 4.2. 用户体验 (UX) Bug

#### **评估卡片状态不更新 (中优先级)**
- **现象**: Dashboard 上各评估卡片右上角的设备在线状态（如 0/4）始终不更新。
- **现象**: `Dashboard.jsx` 中，卡片状态的更新逻辑依赖于 `deviceOnlineMap`。问题可能出在组件的重渲染逻辑上，或者 `deviceOnlineMap` 的状态更新未能正确触发卡片组件的刷新。
- **建议**: **前端**: 调试 `Dashboard.jsx` 中 `ASSESSMENTS.map` 部分，确保在 `deviceOnlineMap` 变化时，每个卡片都能被正确重新渲染。

#### **连接状态反馈矛盾 (中优先级)**
- **现象**: 点击“一键连接”，按钮文本立刻变为“已连接”，但设备在线数显示为 0/7。
- **根因**: 前端 `connectAllDevices` 函数在调用 `connPort` 后立即将状态设置为 `connected`，而未等待设备识别完成。
- **建议**: **前端**: 将 `setDeviceConnStatus('connected')` 的调用时机，从 `connPort` 调用后，移至收到第一个 `deviceStatus` WebSocket 消息或一个定时器之后，以提供更准确的用户反馈。

#### **握力页面状态显示不佳 (低优先级)**
- **现象**: 握力评估页面右上角显示“未知”。
- **根因**: `GripAssessment.jsx` 页面未能正确获取并显示当前连接的设备信息。
- **建议**: **前端**: 在 `GripAssessment.jsx` 中，从 `useAssessment` 上下文获取 `deviceOnlineMap`，并根据 `HL` 和 `HR` 的在线状态来更新右上角的 UI 显示。
