## 二、发现并修复的Bug

### Bug #1: matrix表schema缺失列（严重）

**问题描述**: `init.db` 和 `foot.db` 中的 `matrix` 表缺少 `timestamp` 和 `select` 两个列，但后端 `storageData` 函数的 INSERT 语句尝试向这两个列插入数据。

**影响**: 所有评估的采集数据**完全无法写入数据库**，导致：
- CSV导出功能失效（无数据可导出）
- 数据库回放功能失效（无数据可回放）
- 后端报告生成API失效（无数据可分析）
- `getColHistory` API超时（查询空表但可能触发锁等待）

**修复内容**:
1. 为 `init.db` 和 `foot.db` 的 matrix 表添加 `timestamp TEXT` 和 `select TEXT` 列
2. 修复 `ensureMatrixNameColumn` 函数，增加对 `timestamp` 和 `select` 列的自动检查和修复

**修复文件**: `back-end/code/server/serialServer.js`, `back-end/code/db/init.db`, `back-end/code/db/foot.db`

## 三、唯一失败用例分析

### T43 - 生成步态报告 (getFootPdf)

**失败原因**: `getFootPdf` API 要求 matrix 表中包含 `foot1, foot2, foot3, foot4` 四路脚垫数据，但当前虚拟串口模拟器只提供了一路脚垫数据（foot1）。

**分析**: 这不是应用Bug，而是测试数据不完整。步态评估需要4块脚垫板同时采集数据，用户提供的原始数据中只有1个脚垫CSV文件。

**建议**: 如果需要完整测试步态报告生成，需要提供4块脚垫板的采集数据文件。
